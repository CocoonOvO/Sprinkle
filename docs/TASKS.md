## 📋 开发流程（必须遵循）

### 整体流程

```
┌─────────────────────────────────────────────────────────────┐
│ 1. Phase 开发循环（每个 Phase）                              │
│    ├── 阅读准备 + 设计文档                                   │
│    ├── 开发（严格遵循设计文档）                              │
│    └── 自测完善 → 合并到本地 develop 分支                    │
│                        ↓                                     │
│    重复直到所有 Phase 完成                                  │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. 集成测试（develop 分支，所有 Phase 完成后）               │
│    ├── 设计测试用例（基于实际使用场景）                      │
│    ├── 逐个执行测试                                         │
│    └── 验证结果准确性（不能只看不报错）                      │
│                        ↓                                     │
│    测试通过？                                               │
│    ├── 否 → 更新任务文档 → 回到 Phase 开发循环              │
│    └── 是 → 进入下一步                                      │
└─────────────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. 提交审批                                                │
│    ├── 司康统一提交 PR → 布莱妮审批                         │
│    └── 布莱妮打回 → 回到 Phase 开发循环                     │
└─────────────────────────────────────────────────────────────┘
```

### Phase 标准流程

```
1. 📖 阅读准备
   ├── 阅读 ARCHITECTURE.md（确认设计）
   ├── 阅读 TASKS.md（确认任务要求）
   └── 阅读依赖模块的代码和接口

2. 📐 设计阶段
   ├── 生成设计文档（docs/design/phaseX-design.md）
   ├── 确认符合项目规范
   └── 确认与其他模块对接

3. 💻 开发阶段
   └── 严格遵循设计文档开发

4. ✅ 检查阶段
   ├── 检查代码是否符合设计文档
   ├── 检查是否符合项目规范
   └── 自测完善后合并到 develop 分支
```

### 集成测试要求（非常重要）

**必须在所有 Phase 合并到 develop 后进行**

1. **设计测试用例**
   - 基于实际使用场景
   - 覆盖正常流程、边界条件、异常流程
   - 测试用例要全面

2. **执行测试**
   - 逐个执行测试用例
   - 记录测试结果

3. **验证结果准确性**
   - 不能只看不报错
   - 必须验证结果是否符合预期

4. **问题处理**
   - 如果有问题，更新任务管理文档
   - 拆分安排下一轮任务解决问题
   - 继续循环直到集成测试通过

### PR 审批流程

1. **司康统一提交 PR**（不是 subagent）
2. **布莱妮审批**
3. **如果打回**：回到 Phase 开发循环，解决问题

---

**循环终止条件**：布莱妮审批通过
